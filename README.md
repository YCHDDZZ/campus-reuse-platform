# 🎓 校园二手闲置循环平台 - AI驱动全周期开发实践

**毕业设计课题**：基于AI Agent协作的校园二手闲置循环平台快速交付模式研究与实践

## 🎯 项目背景与核心痛点

校园二手交易市场长期存在**信息零散、流程不规范、信任机制缺失**等问题。同时，个人开发者在面对Spring Boot、Vue3、Redis、MinIO等多技术栈集成时，开发效率极低。本项目通过构建AI驱动的协作体系，实现了从需求分析到部署监控的全周期自动化开发。

## 🚀 AI驱动协作体系架构

### 长链推理流水线
本项目采用"需求分析→架构设计→编码实施→测试验证→部署监控"的完整AI驱动开发流程：

#### 1. **需求智能分解**
- 利用大语言模型Agent对功能需求进行结构化分解
- 自动生成UML用例图和业务流程图初稿
- 智能识别业务逻辑边界和接口规范

#### 2. **多Agent协作开发**
- **架构Agent**：生成数据库ER图并校验范式，确保数据一致性
- **编码Agent**：基于Spring Boot + MyBatis-Plus模式自动生成实体类、Mapper接口及RESTful API骨架
- **测试Agent**：根据需求自动构建黑盒测试用例并执行接口压力测试

#### 3. **自动化部署流水线**
- Git Hook自动同步至开发环境
- 持续集成/持续部署(CI/CD)流程
- 实时性能监控和异常告警

## 🚀 快速开始

### 环境要求
- **JDK 11** - Java开发环境
- **Maven 3.9+** - 项目构建工具
- **Node.js 20+** - JavaScript运行环境
- **npm 10+** - 包管理器
- **MySQL 8** - 数据库服务
- **Redis 6/7** - 缓存服务（可选）
- **MinIO** - 对象存储服务

### 数据库初始化
```sql
-- 执行全量建表脚本
source backend/sql/schema-mysql.sql

-- 已有数据库执行增量脚本
source backend/sql/alter-platform-user-add-avatar-url.sql
source backend/sql/optimize-indexes.sql
```

### 后端启动
```bash
cd backend
mvn spring-boot:run
```
默认访问地址：`http://127.0.0.1:8080`

### 前端启动
```bash
cd frontend
npm install
npm run dev
```
默认访问地址：`http://127.0.0.1:5173`

## 🔧 关键技术特性

### 智能缓存策略
- Redis可用时：使用分布式缓存提升性能
- Redis不可用：自动降级为本地内存缓存，确保系统可用性

### 文件存储管理
- MinIO集成：支持商品图片、二维码、头像等文件上传
- 文件预览与删除功能
- S3 API兼容性设计

### 安全机制
- JWT令牌认证
- 权限细粒度控制
- 数据加密传输
